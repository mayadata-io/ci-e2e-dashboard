CI/E2E Portal Overview

Workloads-Dashboard

The successful builds of OpenEBS from master are deployed onto various live deployments that are hosted on Kubernetes cluster deployed currently on GKE. These reference stateful workloads are deployed for both the storage engines Jiva and cStor.

CI/E2E dashboard:

This CI/E2E portal is built to be used both by OpenEBS developer and user community. Developers and users get the detailed test report across various release branches include the master branch.

Stateful workloads using OpenEBS

This portal also provides resource summary dashboard for each stateful workload.

As shown above the users can see the details (status of each replica and on which node it is running) of both application and OpenEBS pods (controllers & replicas)

Running chaos tests on live deployments:

End users can run ready chaos tests against the live deployment by choosing the test on the portal itself. When the test is run, they can see the status of Kubernetes resources changing accordingly in the real time and how OpenEBS continues to provide data availability to the stateful application.
